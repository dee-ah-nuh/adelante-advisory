If creating visual artifacts (slides, mocks, throwaway prototypes, etc), copy assets out of `assets/` and `ui_kits/` and create static HTML files for the user to view. If working on production code, you can copy assets and read the rules here to become an expert in designing with this brand.

Key things to internalize before designing:
- **"Editorial Warmth"** — magazine/letter feel, not SaaS. Warm paper background `#F5F1EA`, deep forest ink `#1F2B23`, terracotta accent `#C45A3B` used sparingly.
- **Three fonts, three jobs** — Instrument Serif (display, italic), Inter Tight (body/UI), JetBrains Mono (eyebrows only, uppercase + letter-spaced).
- **Hero treatment** — two regular lines + one italic line + one terracotta word is the brand's signature type move.
- **No gradients, no emoji, no icon fonts, no rounded pill buttons.** Radius = 4px everywhere.
- **Bilingual EN/ES are equal citizens** — copy both when creating marketing surfaces.

If the user invokes this skill without any other guidance, ask them what they want to build or design, ask a few clarifying questions (audience, EN or ES or both, which surface / screen, what level of fidelity), and act as an expert designer who outputs HTML artifacts *or* production code, depending on the need.
